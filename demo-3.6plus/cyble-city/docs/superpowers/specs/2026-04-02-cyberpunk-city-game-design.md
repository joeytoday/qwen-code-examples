# 赛博朋克城市飞行器 - 设计文档

**日期**: 2026-04-02  
**类型**: 3D 飞行探索游戏  
**技术栈**: Three.js + 单 HTML 文件

---

## 1. 核心概念

### 游戏类型
3D 自由飞行探索游戏，玩家在赛博朋克风格的城市中自由穿梭，收集发光能量球。

### 核心玩法
- **探索**: 使用键盘方向键在 3D 城市中自由飞行
- **收集**: 拾取漂浮的能量球获得分数
- **沉浸**: 无时间限制、无失败条件，纯粹体验赛博朋克美学

---

## 2. 技术架构

### 渲染引擎
- **Three.js** (通过 CDN 引入)
- 真正的 WebGL 3D 渲染
- 支持光影、雾效、粒子效果

单个 HTML 文件，直接在浏览器打开即可运行，无需构建工具。

## 3. 视觉设计

### 城市环境
| 元素 | 描述 |
|------|------|
| 建筑数量 | 10-15 栋 |
| 建筑风格 | 高低错落的摩天大楼，部分带有退台设计 |
| 霓虹元素 | 建筑边缘青色/粉色光条、发光窗户网格、全息广告牌 |
| 地面 | 深色网格地面，带发光线条 |
| 氛围 | 轻微紫色雾气增强深度感 |

### 配色方案
```css
背景色：#0a0a0f (深紫黑)
主霓虹：#00f3ff (青色)
辅霓虹：#ff00aa (粉色)
点缀色：#aa00ff (紫色)
建筑主体：#1a1a2e (深蓝灰)
```

### 飞行器设计
- **造型**: 由发光线条构成的三角翼抽象造型
- **材质**: 霓虹光条效果，边缘发光
- **尾焰**: 动态粒子推进器效果，青色渐变
- **尺寸**: 相对于建筑的合适比例

### 能量球设计
- **外观**: 半透明发光球体，内部有旋转粒子
- **颜色**: 金色/青色随机
- **大小**: 直径约飞行器的 1/3
- **分布**: 随机散布在城市空间中，高度不一

## 4. 交互设计

### 操控方式
| 按键 | 功能 |
|------|------|
| ↑ | 向前飞行 |
| ↓ | 向后飞行 |
| ← | 向左平移 |
| → | 向右平移 |
| Shift | 加速（按住时速度 x2） |

### 相机系统
- **第三人称跟随**: 相机位于飞行器后上方
- **平滑跟随**: 使用 LERP 插值实现流畅的相机移动
- **视角**: 略微俯视，能看到前方城市景观

### UI 界面
- **分数显示**: 左上角，霓虹字体
- **操作提示**: 右下角半透明面板，显示按键说明
- **收集反馈**: 收集能量球时短暂闪光 + 数字飘字效果

## 5. 核心组件

### 5.1 SceneManager
负责初始化 Three.js 场景、相机、渲染器。

### 5.2 CityGenerator
程序化生成城市建筑：
- 随机高度（20-80 单位）
- 随机位置（避免重叠）
- 添加霓虹装饰（边缘光条、窗户网格）

### 5.3 PlayerController
处理玩家输入和飞行器运动：
- 监听键盘事件
- 计算速度和位置
- 边界限制（防止飞出城市）

### 5.4 CollectibleManager
管理能量球的生成和收集：
- 初始生成 20 个能量球
- 收集后重新生成到新位置
- 检测碰撞并更新分数

### 5.5 ParticleSystem
视觉效果：
- 飞行器尾焰粒子
- 能量球收集时的爆炸效果
- 城市氛围粒子（可选）

---

## 6. 数据流

```
用户输入 (键盘) 
    ↓
PlayerController 更新飞行器位置
    ↓
相机平滑跟随飞行器
    ↓
CollectibleManager 检测碰撞
    ↓
更新分数 UI + 播放反馈效果
```

## 7. 性能考虑

### 优化策略
- 建筑使用简单几何体（BoxGeometry），减少面数
- 能量球使用 Sprite 或低多边形 Sphere
- 粒子系统限制最大数量
- 使用 Fog 遮挡远处建筑，减少渲染负担

### 目标帧率
- 桌面端：60 FPS
- 移动端：30+ FPS（如果支持触摸）
