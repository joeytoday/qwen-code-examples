# Bolt.new 产品调研 - 工作区页面分析

## 一、页面概览

**页面类型**：工作区（Workspace）  
**URL 格式**：bolt.new/~/[project-id]  
**布局结构**：三栏布局（左侧对话面板 + 中间代码编辑器 + 右侧预览/设置面板）  
**主题色**：深色主题（黑色/深灰色背景）

## 四、中间代码编辑区

### 4.1 文件树面板

#### 顶部工具栏
**按钮**（从左到右）：
- `📁 Files` - 文件树标题
- `🔍 Search` - 搜索按钮

**搜索功能**：
- 点击 Search 按钮展开搜索框
- 支持文件名搜索
- 实时过滤文件列表

#### 文件树结构

**显示内容**：
```
.bolt
config.json
prompt
src/
  .env
  .gitignore
  eslint.config.js
  index.html
  package-lock.json
  package.json
  postcss.config.js
  tailwind.config.js
  tsconfig.app.json
  tsconfig.json
  tsconfig.node.json
  vite.config.ts
```

**特点**：
1. **文件夹折叠/展开**：
   - 点击文件夹名称或箭头图标
   - 展开显示子文件
   - 折叠隐藏子文件

2. **文件图标**：
   - 不同文件类型显示不同图标
   - 如：JS 文件显示 JS 图标
   - 配置文件显示对应图标

3. **当前文件高亮**：
   - 当前打开的文件背景色高亮
   - 如：`index.html` 显示蓝色背景

4. **文件搜索高亮**：
   - 搜索时匹配的文件名高亮显示
   - 如：搜索 "app" 时，App.css、App.tsx 等高亮

### 4.2 代码编辑器

#### 编辑器特性

**1. 语法高亮**
- 支持多种语言（HTML、CSS、JS、TS、JSON 等）
- 关键字、标签、属性等不同颜色显示
- 如：HTML 标签为绿色，属性为蓝色

**2. 行号显示**
- 左侧显示行号（1, 2, 3...）
- 方便定位代码位置

**3. 代码折叠**
- 支持折叠代码块
- 点击行号左侧的折叠图标
- 如：`<head>` 标签可以折叠

**4. 文件标签页**
- 顶部显示当前文件名：`index.html`
- 支持多文件切换（如果打开多个文件）

**5. 代码搜索高亮**
- 搜索关键词时高亮显示
- 如：搜索 "App" 时，代码中的 "App" 高亮

#### 编辑器内容示例

```html
<!doctype html>
<html lang="en">
  <head>
    <meta charset="UTF-8" />
    <link rel="icon" type="image/svg+xml" href="/vite.svg" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <title>Document Enrichment</title>
    <meta property="og:image" content="https://bolt.new/static/og_default.png" />
    <meta name="twitter:card" content="summary_large_image" />
    <meta name="twitter:image" content="https://bolt.new/static/og_default.png" />
  </head>
  <body>
    <div id="root"></div>
    <script type="module" src="/src/main.tsx"></script>
  </body>
</html>
```

### 4.3 面板拖拽调整

**功能**：
- 文件树和代码编辑器之间有分割线
- 鼠标悬停在分割线上时，光标变为调整大小图标
- 拖拽分割线可以调整两个面板的宽度
- 可以将文件树拖窄，给代码编辑器更多空间

**交互**：
1. 鼠标悬停在分割线上
2. 光标变为 `↔` 图标
3. 按住鼠标左键拖拽
4. 释放鼠标完成调整

## 七、交互设计

### 7.1 对话交互

**流程**：
1. 用户在输入框输入需求
2. 点击发送按钮或按 Enter
3. AI 生成 Plan（计划）
4. AI 执行 Plan 中的任务
5. 实时更新文件和代码
6. 显示完成总结

**Plan 执行过程**：
- 任务列表逐个执行
- 已完成的任务显示 ✓
- 正在执行的任务高亮
- 最后显示 "Plan completed"

### 7.2 代码编辑交互

**流程**：
1. 在文件树中点击文件
2. 代码编辑器显示文件内容
3. AI 自动编辑代码（如果有任务）
4. 实时显示代码变更
5. 保存并更新预览

**代码变更动画**：
- 新增代码：绿色高亮
- 删除代码：红色高亮
- 修改代码：黄色高亮

### 7.3 预览交互

**流程**：
1. 代码变更后自动刷新预览
2. 支持在预览中交互操作
3. 支持页面导航
4. 支持响应式预览

**刷新机制**：
- 代码保存后自动刷新
- 支持手动刷新按钮
- 支持热重载（HMR）

### 7.4 文件搜索交互

**流程**：
1. 点击 Search 按钮
2. 输入搜索关键词
3. 实时过滤文件列表
4. 高亮匹配的文件名
5. 点击文件打开

**搜索特点**：
- 支持模糊搜索
- 支持文件名搜索
- 支持文件内容搜索（可能）

## 十、用户体验

### 10.1 工作流程

**典型工作流程**：
```
1. 用户描述需求
   ↓
2. AI 生成 Plan
   ↓
3. AI 执行任务（创建/编辑文件）
   ↓
4. 实时预览效果
   ↓
5. 用户查看并反馈
   ↓
6. 继续迭代优化
   ↓
7. 发布项目
```

### 10.2 信息层级

**重要性排序**：
1. **最重要**：代码编辑器和预览（视觉中心）
2. **次要**：对话面板（左侧）
3. **辅助**：底部终端和日志

### 10.3 快捷操作
