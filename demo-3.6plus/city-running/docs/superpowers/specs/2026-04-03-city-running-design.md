# City Running - 城市跑步游戏设计文档

**日期**: 2026-04-03  
**类型**: 游戏设计  
**风格**: Tile Pack City 3 低多边形鲜艳色彩

## 1. 游戏概述

### 1.1 核心概念
第三人称城市跑步游戏，人物在预设的城市街道上自动奔跑，玩家控制左右移动和跳跃，收集能量物品。

### 1.2 游戏类型
- **视角**: 第三人称追尾视角
- **玩法**: 自动前进 + 左右移动 + 跳跃
- **目标**: 收集能量物品，获得高分
- **风格**: Tile Pack City 3 低多边形鲜艳色彩

### 1.3 技术栈
- **引擎**: Three.js r163
- **语言**: JavaScript (单文件)
- **依赖**: 零依赖（Three.js 通过 CDN 加载）
- **文件**: `index.html` + `game.js`

## 2. 城市设计

### 2.1 布局结构

**道路网络**:
- **主干道**: 一条贯穿城市的宽阔大道（约 200 单位长，15-20 单位宽），作为主要奔跑路线
- **横向街道**: 3-4 条横向道路与主干道交叉，形成街区网络
- **人行道**: 道路两侧有人行道区域（浅灰色），放置街道家具

**建筑区块**:
- 每个街区内布置不同类型的建筑
- 建筑之间留有适当间距
- 建筑高度多样化（2-8 层）

**中央公园**:
- 城市中心区域设置开放空间
- 布置 12 棵树木形成绿地
- 可作为视觉焦点和跑步路线转折点

### 2.2 城市元素清单

**建筑类型**（10 种，复用现有代码）:
1. 商店 (Shop) - 小型店铺带遮阳棚
2. 公寓 (Apartment) - 2-4 层住宅带阳台
3. 办公楼 (Office) - 玻璃幕墙建筑
4. 医院 (Hospital) - 白色建筑带红十字
5. 消防站 (FireStation) - 红色建筑带车库门
6. 古典建筑 (Classical) - 带柱廊和三角楣
7. 教堂 (Church) - 带圆顶和尖塔
8. 摩天大楼 (Skyscraper) - 5-8 层扭曲塔楼
9. 体育场 (Stadium) - 椭圆形带环形屋顶
10. 商场 (Mall) - 大型盒子建筑带玻璃穹顶

**街道家具**（11 种）:
- 路灯、树木、长椅、垃圾桶、邮箱、消防栓、自行车架、花坛、交通信号灯、路牌

**装饰车辆**（3 种）:
- 出租车、警车、公交车（停放在路边）

### 2.3 比例关系

| 元素 | 尺寸 | 说明 |
|------|------|------|
| 人物高度 | ~2 单位 | Soldier.glb 模型 |
| 道路宽度 | 15-20 单位 | 足够人物奔跑 |
| 人行道宽度 | 3-5 单位 | 道路两侧 |
| 建筑高度 | 10-40 单位 | 2-8 层不等 |
| 建筑宽度 | 8-15 单位 | 根据类型变化 |
| 城市总尺寸 | 400x400 单位 | 包含所有元素 |
| 树木高度 | 4-6 单位 | 公园和街道绿化 |

### 2.4 色彩方案

**Tile Pack City 3 调色板**:
- 道路: `#4a4a4a` (深灰色)
- 人行道: `#b0b0b0` (浅灰色)
- 草地: `#7cb342` (鲜绿色)
- 天空: `#87ceeb` (浅蓝色)
- 建筑基础: 白色 `#ffffff` 或浅灰色
- 建筑屋顶: 多样化颜色（红、蓝、绿、棕等）
- 收集物: `#ffd700` (金色) 带发光效果

## 3. 游戏机制

### 3.1 控制方式

| 按键 | 动作 | 说明 |
|------|------|------|
| A / ← | 向左移动 | 横向移动 |
| D / → | 向右移动 | 横向移动 |
| SPACE / ↑ | 跳跃 | 越过障碍物或获取高处收集物 |
| R | 重置位置 | 如果跑偏或卡住 |
| C | 切换相机 | 追尾/头顶/侧视三种视角 |
| ESC | 暂停 | 暂停/恢复游戏 |

### 3.2 物理系统

**跑步参数**:
- 自动前进速度: 5 m/s (基础)
- 冲刺速度: 7.5 m/s (按住 Shift)
- 横向移动速度: 3 m/s
- 跳跃高度: 2 单位
- 重力: 9.8 m/s²
- 跳跃持续时间: 约 0.6 秒

**碰撞检测**:
- 道路边界: 人物不能离开道路
- 障碍物: 锥形桶、路障等需要躲避
- 收集物: 接触即收集

### 3.3 收集系统

**能量球**:
- 外观: 发光球体（金色/蓝色）
- 分布: 沿道路随机散布
- 效果: 收集后 +10 分
- 特效: 收集时播放粒子效果

**特殊收集物**:
- 位置: 需要跳跃才能获取的高处
- 效果: +50 分
- 外观: 更大的发光球体带旋转动画

### 3.4 游戏流程

1. **开始界面**: 游戏标题 + "开始奔跑" 按钮
2. **游戏进行**: 人物自动奔跑，玩家控制移动和跳跃
3. **分数显示**: 实时更新 HUD
4. **暂停功能**: ESC 键暂停
5. **游戏结束**: 无明确结束，持续奔跑收集

## 4. 技术架构

### 4.1 文件结构

```

├── game.js             # 游戏逻辑（约 1500 行）
├── README.md           # 项目文档
└── .gitignore          # Git 忽略文件
```

### 4.2 代码模块

#### SceneManager
- 场景初始化
- 光照设置（环境光 + 方向光）
- 雾效配置
- 天空盒/背景色

#### CityGenerator
- `createRoadNetwork()`: 创建道路系统
  - 主干道（直线大道）
  - 横向街道（交叉路）
  - 人行道（道路两侧）
- `placeBuildings()`: 布置建筑
  - 遍历街区网格
  - 随机选择建筑类型
  - 放置并旋转建筑
- `addStreetFurniture()`: 添加街道装饰
  - 路灯（沿道路等距分布）
  - 树木（人行道和公园）
  - 长椅、垃圾桶等
- `addVehicles()`: 放置装饰车辆
- `createCentralPark()`: 创建中央公园

#### PlayerController
- 复用 Soldier.glb 模型加载
- 复用动画混合系统（idle/walk/run）
- 新写跑步控制逻辑:
  - 自动前进（持续向前移动）
  - 横向移动（A/D 键控制）
  - 跳跃（SPACE 键，带重力模拟）
  - 道路边界限制

#### CollectibleManager
- 生成能量球（沿道路随机位置）
- 检测碰撞（人物与收集物接触）
- 播放收集特效（粒子效果 + 音效占位）
- 更新分数

#### CameraController
- 第三人称追尾相机（默认）
- 头顶视角（按 C 切换）
- 侧视角（按 C 循环切换）
- 平滑跟随（lerp 插值）

## 5. 视觉效果

### 5.1 渲染设置
- 抗锯齿: 启用

- 雾效: 轻微雾效增加深度感

- 人物: 跑步动画（循环播放）
- 收集物: 旋转 + 上下浮动动画
- 收集特效: 粒子爆发效果

### 5.3 相机视角
**默认视角**（第三人称追尾）:
- 位置: 人物后上方 45 度角

**备选视角**（按 C 切换）:
- 头顶视角: 正上方垂直向下

## 9. 风险和缓解

### 9.1 技术风险
- **风险**: Soldier 模型加载失败
  - **缓解**: 使用 CDN 备用链接，添加加载错误处理
- **风险**: 相机跟随不平滑
  - **缓解**: 使用 lerp 插值，调整跟随参数

### 9.2 设计风险
- **风险**: 城市布局不够完整
  - **缓解**: 预设固定布局，确保建筑密度
- **风险**: 人物比例不协调
  - **缓解**: 严格按照比例表调整建筑和道路尺寸
